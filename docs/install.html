zigbee — install & first-run guide

From a clean Linux machine to your first SWAP-paid retrieval against a public Sepolia bootnode. No prior Swarm experience assumed.

What is this thing, in one paragraph

Swarm is a peer-to-peer storage network — files are chopped into 4 KB chunks, scattered across thousands of participating nodes, and each chunk is addressed by a hash. The reference node software, written in Go, is called bee. zigbee is a separate much smaller Swarm client, written in Zig, that joins the network and pulls files out of it. Unlike bee it doesn't store data and doesn't try to be a full node — it's a retrieval-only ultra-light client (~6 MB binary) designed for IoT / embedded / low-resource hosts.

In this guide you'll install zigbee, fund a small testnet wallet, deploy a Swarm chequebook contract on the Ethereum Sepolia testnet, and run zigbee against a public Sepolia bootnode so it discovers an optimal peer set and pays them with real EIP-712-signed cheques while retrieving files.

Bootnode vs. local bee — what's the difference? This guide uses the bootnode path: zigbee dials a well-known Sepolia bootnode, and via the Swarm hive protocol it receives a list of other peers and dials them too. With --max-peers 20 zigbee ends up connected to ~20 real testnet nodes, sorted by XOR-proximity to whatever you're retrieving. That's how you get optimal-peer routing.

The alternative — running a local bee yourself and pointing zigbee at 127.0.0.1:1634 — is just the developer-loop setup. Useful for testing uploads (you need bee to upload). Not what you want for actual retrieval. We cover it in Appendix A at the end.

Step 1. Prerequisites

You need these installed before anything below will work. Most likely you already have several of them.

ToolWhyInstall hint
git Cloning source code. Already on most systems. sudo apt install git
zig 0.15.x Compiles zigbee. Download from ziglang.org/download — extract the tarball, put zig on your PATH. Verify with zig version (should print 0.15.x).
cast (foundry) Sends the Sepolia transactions that deploy your chequebook. curl -L https://foundry.paradigm.xyz | bash && foundryup — gives you cast, forge, anvil. We only use cast.
curl, python3, bash, xxd Used by the helper script that deploys the chequebook. Standard on every Linux/macOS. xxd ships with vim-common; sudo apt install xxd if missing.

Quick check that you have everything:

git --version
zig version          # expect 0.15.x
cast --version       # foundry
curl --version
python3 --version
xxd -v

You do NOT need go to follow the bootnode path — Go is only needed to build a local bee binary, which we don't use here. (See Appendix A if you want to.)

Step 2. Get the source code ~2 min

Pick a working directory. Throughout this guide we use ~/swarm/, but anywhere is fine — adjust paths as you go.

mkdir -p ~/swarm && cd ~/swarm
git clone https://github.com/martinconic/zigbee.git
cd zigbee

The repo ships with an install kit at examples/install-kit/ — four numbered scripts that walk you through the rest of this guide. They auto-detect the zigbee binary's location and accept config overrides via a config.sh file. We'll use them throughout.

The install kit at a glance 
examples/install-kit/
├── README.md                  # script reference
├── config.example.sh          # template; copy to config.sh
├── _common.sh                 # sourced by every script
├── 1-check-funds.sh           # verify Sepolia ETH + sBZZ
├── 2-deploy-chequebook.sh     # one-time chain step
├── 3-run-zigbee.sh            # start daemon (foreground)
└── 4-retrieve.sh              # curl /bytes/<ref> helper

The kit reads config.sh if present (cp config.example.sh config.sh && $EDITOR config.sh) — for things like a private RPC URL, a different FUND_AMOUNT_BZZ, or a custom bootnode. config.sh is gitignored, so your overrides never get committed. Defaults work out-of-the-box on Sepolia testnet, so you can skip this step on a first run.

Step 3. Build zigbee ~30 s

cd ~/swarm/zigbee
zig build -Doptimize=ReleaseSafe

Output binary is at ~/swarm/zigbee/zig-out/bin/zigbee (~6 MB). Verify it runs:

./zig-out/bin/zigbee --help

Step 4. Generate your zigbee identity ~1 s

Zigbee needs a long-lived secp256k1 keypair — same kind Ethereum uses. Running the identity subcommand creates one (or prints the existing one if you already have it):

cd ~/swarm/zigbee
./zig-out/bin/zigbee identity

You'll see something like:

eth_address=0x<your-20-byte-eth-address>
overlay=0x<your-32-byte-overlay>
network_id=10

What this means:

  • eth_address — this is the address you'll fund in the next two steps. It's a regular Ethereum address derived from your zigbee key. Write it down.
  • overlay — your peer ID in the Swarm network (derived from eth_address + a per-network nonce). You don't need to copy it; it'll show up in logs.
  • network_id=10 — the Sepolia testnet network ID.

The key is saved at ~/.zigbee/identity.key — a 64-byte file (32-byte private key ‖ 32-byte bzz nonce). Back it up; if you lose it you lose access to your chequebook.

Treat ~/.zigbee/identity.key like a wallet private key — because it is one. Anyone with this file can spend any sBZZ in your chequebook. We're on testnet so nothing is real money, but build the habit now.

Step 5. FUND Get Sepolia ETH for gas ~5–10 min

Deploying your chequebook is a real on-chain transaction on the Sepolia Ethereum testnet, and on-chain transactions cost gas. Gas is paid in (testnet) ETH. You need about 0.005 ETH on Sepolia — call it 0.01 to be safe.

Pick any Sepolia faucet. Common choices:

  • alchemy.com/faucets/ethereum-sepolia — drips 0.05 ETH/day with a free Alchemy account.
  • sepolia-faucet.pk910.de — proof-of-work captcha, no signup. Slow but reliable.
  • infura.io/faucet/sepolia — needs an Infura account.

Paste your eth_address from step 4. Wait for the transaction to confirm (a minute or two), then check both this and the next funding step in one go:

cd ~/swarm/zigbee
./examples/install-kit/1-check-funds.sh

This reads your identity, derives the eth address, queries Sepolia for both ETH and sBZZ balances, and prints OK / insufficient with faucet links. No transactions sent. If ETH is missing the output points you back to the faucets above; if sBZZ is missing, see the next step.

Step 6. FUND Get Sepolia (s)BZZ ~5–15 min

BZZ is Swarm's native ERC-20 token. There's a real BZZ on Ethereum mainnet, and a separate testnet BZZ on Sepolia (sometimes called sBZZ or gBZZ — same contract, just on Sepolia). You need it to fund your chequebook: the chequebook contract holds a sBZZ balance, and your cheques draw against that balance.

Aim for at least 1 sBZZ on Sepolia in your eth_address (more if you'll be retrieving lots of data — each ~3 chunks costs about 1×1012 sub-units of payout delta, so 1 sBZZ = 1016 sub-units covers thousands of chunks).

Where to get sBZZ. There isn't a single canonical public faucet; the easiest path is to ask in the Swarm Discord (the #dev channel) — community members and team occasionally drip sBZZ to developers building on testnet. The contract address is 0x543dDb01Ba47acB11de34891cD86B675F04840db on Sepolia (16 decimals).

Once you've received it, re-run the funds check from step 5:

./examples/install-kit/1-check-funds.sh

Both lines should now print "OK". If sBZZ still shows insufficient, the chain hasn't propagated yet — wait a minute and re-run.

Step 7. Deploy your chequebook on Sepolia ~30 s

This is the one-time chain step. Run it once per zigbee identity; the result lasts forever (or until you switch testnets). Note: it talks directly to a public Sepolia RPC — no local bee needs to be running.

cd ~/swarm/zigbee
./examples/install-kit/2-deploy-chequebook.sh

What this does, in plain English:

  1. Reads your ~/.zigbee/identity.key to get the private key and derives the eth address.
  2. Calls the canonical Swarm chequebook factory contract on Sepolia (0x0fF044F6bB4F684a5A149B46D7eC03ea659F98A1). The factory deploys a new SimpleSwap contract on your behalf, owned by your eth address. (This is your chequebook.)
  3. Reads the freshly-deployed contract address from the transaction receipt.
  4. Calls transfer() on the sBZZ token to send 1 sBZZ from your eth address into the chequebook.
  5. Writes ~/.zigbee/chequebook.json — the credential file zigbee will load on startup.

You'll see the chequebook contract address in the output, like:

[06-deploy] issuer eth address: 0x<your-eth-address>
[06-deploy] factory 0x0fF0…98A1 → gBZZ token: 0x543dDb01Ba47acB11de34891cD86B675F04840db (decimals=16)
[06-deploy] funding amount: 1 BZZ = 10000000000000000 raw units
[06-deploy] deploy tx: 0x<tx-hash>
[06-deploy] chequebook deployed at: 0x<your-chequebook-address>
[06-deploy] funded with 1 BZZ
[06-deploy] wrote ~/.zigbee/chequebook.json

From now on zigbee can sign cheques drawn against that chequebook.

Backup: ~/.zigbee/chequebook.json — this file plus ~/.zigbee/identity.key together let you recover from a wiped disk. Copy them somewhere safe. In production this is the entire payload an operator would flash onto a deployed device alongside firmware.

Step 8. OPTIONAL Inspect the bootnode list

Bootnodes are well-known public peers run by the Swarm Foundation that anyone can dial to enter the network. The Sepolia testnet publishes its bootnode list at the DNS name sepolia.testnet.ethswarm.org as /dnsaddr TXT records.

You don't have to do anything here — the next step launches zigbee with --bootnode /dnsaddr/sepolia.testnet.ethswarm.org which does the lookup internally and tries each candidate in turn. But if you want to see the list (handy for sanity-checking your DNS or picking a specific IP for --peer):

./zig-out/bin/zigbee resolve sepolia.testnet.ethswarm.org

You'll see something like:

resolved 4 multiaddrs for sepolia.testnet.ethswarm.org:
  /ip4/167.235.96.31/tcp/32491/p2p/16Uiu2HAm…
  /ip4/116.202.130.19/tcp/30634/p2p/16Uiu2HAm…
  /ip4/95.216.91.90/tcp/30634/p2p/16Uiu2HAm…
  /ip4/142.132.224.135/tcp/30634/p2p/16Uiu2HAm…

Step 9. Run the zigbee daemon ~10 s

Now the main event — start zigbee in daemon mode against the Sepolia bootnodes, with the chequebook credential loaded:

./examples/install-kit/3-run-zigbee.sh

This wraps:

./zig-out/bin/zigbee \
    --bootnode /dnsaddr/sepolia.testnet.ethswarm.org \
    --network-id 10 \
    --chequebook ~/.zigbee/chequebook.json \
    daemon \
    --max-peers 20 \
    --api-port 9090

What each flag means:

  • --bootnode /dnsaddr/sepolia.testnet.ethswarm.org — DNS-resolved bootnode list. zigbee internally walks the candidates in order until one handshakes; the rest stay in the table for hive discovery. (You can override with a fixed /ip4/.../tcp/... multiaddr or use --peer ip:port for a known peer instead — the flags are mutually exclusive.)
  • --network-id 10 — Sepolia testnet network ID. Mismatch and the bootnode's handshake will reject you.
  • --chequebook ~/.zigbee/chequebook.json — load the credential. Without this, zigbee tracks debt with each peer but never pays, so retrieval will stop after about 25 chunks per peer when the peer disconnects you for hitting its accounting threshold.
  • daemon — long-lived process (vs. the one-shot retrieve subcommand).
  • --max-peers 20 — connect to up to 20 peers via hive discovery. More peers = more options for retrieval, since zigbee asks the peer XOR-closest to each chunk.
  • --api-port 9090 — zigbee's local HTTP API.

Zigbee should connect to a bootnode, complete the Noise handshake and the bee handshake protocol, receive a hive broadcast, and start auto-dialing other peers:

[bootnode] resolved 4 candidate(s) from /dnsaddr/sepolia.testnet.ethswarm.org
  1. 167.235.96.31:32491
  2. 116.202.130.19:30634
  …
[daemon] dialing bootnode 1/4 167.235.96.31:32491 (network_id=10)
[connection] noise handshake OK
[bee-handshake] inbound peer_address=7a3c…
[daemon] bootnode handshake done: overlay=7a3c… welcome="…"
[hive] received 41 peer addresses
[dialer] dialing 95.216.91.90:30634 (overlay=44b1…)
…
[host] serving HTTP API on 127.0.0.1:9090

Wait ~10–30 seconds for the auto-dialer to populate the peer set. Then verify in another terminal:

curl -s http://127.0.0.1:9090/peers | python3 -m json.tool

You should see ~10–20 connected peers. The more peers, the better your chances of finding any given chunk on the first attempt.

Leave the zigbee terminal running. Open a second terminal for the next step.

Step 10. Retrieve a file ~1–10 s for small files

You need a known testnet reference (a 64-character hex hash). The easiest way to get one is to ask in the Swarm Discord — someone has almost certainly uploaded a public test file to Sepolia recently.

If you have a reference, fetch it via the kit's retrieve script:

./examples/install-kit/4-retrieve.sh <64-char-hex-ref> /tmp/recovered.bin

The script wraps a timed curl http://127.0.0.1:9090/bytes/<ref> and prints HTTP status + elapsed milliseconds. Or use curl directly:

REF=<paste-the-64-char-hex-here>
curl -o /tmp/recovered.bin "http://127.0.0.1:9090/bytes/$REF"

Watch the zigbee log — for each ~3 chunks fetched you'll see a [swap] … emitting cheque cumulativePayout=… line. That's zigbee paying the peer it's currently retrieving from with an EIP-712-signed cheque. Different chunks may go to different peers; each peer-relationship has its own cumulative payout state at ~/.zigbee/chequebook.state.json.

No reference handy? If you can't get a reference through Discord, the easiest workaround is to follow Appendix A to run a local bee, upload your own random file with bee, then retrieve it via zigbee against the bootnode (the chunks need a minute or so for bee's pushsync to replicate them across the testnet, after which any peer can serve them).

Step 11. Stop and persistent state ~5 s

Send SIGINT to the zigbee daemon (Ctrl-C in its terminal). Zigbee catches the signal, closes its connections cleanly, and exits.

Your data persists across restarts:

  • ~/.zigbee/identity.key — your zigbee identity. Back this up.
  • ~/.zigbee/chequebook.json — chequebook credential. Back this up.
  • ~/.zigbee/chequebook.state.json — per-peer cumulative-payout state. Don't delete this — it tracks how much zigbee has paid each peer, and bee enforces strict monotonicity. Back it up alongside the credential.
  • ~/.zigbee/store/ — LRU chunk cache (default 100 MiB). Safe to delete if you want a clean cache.

Step 12. Glossary & troubleshooting

Glossary

bee The Go reference Swarm node. Speaks the full protocol, joins the network, stores chunks. ~70 MB binary. Bootnodes are bee instances. You don't need to install it for the bootnode path.
zigbee The Zig client we're installing. Ultra-light — retrieval-only as of 0.5 (no upload yet). ~6 MB binary; designed for IoT / embedded.
bootnode A well-known peer with a stable public IP that newcomers can dial to enter the network. Usually run by the project / foundation. Sepolia bootnodes are at sepolia.testnet.ethswarm.org.
hive Swarm's peer-discovery protocol (/swarm/hive/1.1.0/peers). When zigbee connects to a bootnode, the bootnode broadcasts a list of other peers; zigbee dials them up to --max-peers.
overlay A peer's Swarm-network identity — a 32-byte hash derived from its eth address + per-network nonce. zigbee uses XOR distance between overlays to pick which peer to ask for a given chunk.
chunk The unit of storage in Swarm — up to 4 KB of payload, addressed by a hash (CAC) or by signature (SOC).
reference / ref A chunk address, used as the user-facing handle for a file. 64-char hex for unencrypted, 128-char hex for encrypted (the second 64 chars are the symmetric decryption key).
chequebook A SimpleSwap smart contract on the chain that holds your sBZZ. zigbee writes EIP-712-signed cheques against it; the receiving peer can later cash those cheques on-chain.
cheque A signed promise to pay cumulativePayout sBZZ to a specific peer from a specific chequebook. Each new cheque to a given peer must have a strictly higher cumulativePayout than the last (this is what chequebook.state.json tracks per peer).
SWAP The cheque-exchange protocol Swarm uses for accounting. Stands for "Swarm Accounting Protocol." Runs on libp2p stream ID /swarm/swap/1.0.0.
postage stamp / batch A sBZZ-priced authorisation to upload data into Swarm. Required for uploads only — you don't need a stamp to retrieve.
Sepolia An Ethereum testnet — same software as mainnet, worthless test tokens. Swarm's testnet uses Sepolia for chain settlement.
sBZZ / gBZZ The testnet variant of BZZ (Swarm's native ERC-20). Lives at 0x543dDb01Ba47acB11de34891cD86B675F04840db on Sepolia, 16 decimals.

Common problems

SymptomLikely cause / fix
2-deploy-chequebook says "insufficient ETH" The faucet hasn't sent yet, or sent to the wrong address. Re-run ./examples/install-kit/1-check-funds.sh after a few minutes.
2-deploy-chequebook says "insufficient BZZ" You have ETH but no sBZZ. Lower FUND_AMOUNT_BZZ in config.sh (e.g. 0.1) or get more sBZZ.
zigbee resolve returns 0 multiaddrs DNS issue — _dnsaddr.sepolia.testnet.ethswarm.org TXT records aren't reaching you. Try a different network / VPN, or hardcode a known bootnode IP from a teammate.
zigbee logs noise handshake failed The bootnode IP/port is wrong, or the network blocks outbound TCP to that port. Try a different bootnode from the resolve output.
Zigbee starts but /peers stays empty Hive broadcast hasn't arrived yet (give it 30 s) — or the bootnode you picked is currently unhealthy. Try another from the resolve list.
Retrieval returns HTTP 502 None of your connected peers could find the chunk via forwarding. Either the chunk genuinely isn't on the testnet anymore (cleanup happens), or your peer set isn't well-distributed yet — wait for more peers to populate, or restart with a different bootnode.
Some peer log says cheque cumulativePayout is not increasing Your chequebook.state.json got out of sync — likely you wiped it after sending real cheques to that peer. Recover by querying the peer's /chequebook/cheque for your overlay and seeding the state file with that cumulative; see scripts/12-verify-zigbee-swap.sh for the exact code.
cast: command not found Foundry isn't installed or isn't on your PATH. curl -L https://foundry.paradigm.xyz | bash && foundryup then restart your shell.

Appendix A. OPTIONAL Local-bee mode (for uploads or dev)

The bootnode path above gets you retrieval. To upload, you need a local bee — zigbee 0.5 doesn't speak pushsync yet. Run a local bee, upload via its REST API, then either retrieve from your own bee (developer loop) or wait for the network to replicate the chunks and retrieve from the bootnode-discovered peer set.

Tap to expand: full local-bee instructions

A.1 Extra prerequisites

go 1.21+ Compiles bee. sudo apt install golang or go.dev/dl

A.2 Get bee + the rest of the scripts

cd ~/swarm
git clone https://github.com/ethersphere/bee.git
# Drop the rest of the helper scripts (00-, 01-, 02-, 03-, 99-, etc.) into ~/swarm/scripts/
chmod +x scripts/*.sh

Save this as ~/swarm/testnet.yaml:

api-addr: 127.0.0.1:1633
p2p-addr: :1634
network-id: 10
mainnet: false
verbosity: 5
data-dir: ~/.bee/testnet
swap-enable: true
full-node: false
bootnode: /dnsaddr/sepolia.testnet.ethswarm.org
blockchain-rpc-endpoint: https://ethereum-sepolia-rpc.publicnode.com
Adjust script paths. The helper scripts have a ROOT=... baked in pointing at the author's working tree. Edit them to point at your ~/swarm/ tree, or symlink ~/swarm at the author's path if you're really lazy.

A.3 Build bee

cd ~/swarm/bee
go build -o ./bee-bin ./cmd/bee/

A.4 Start bee + buy a stamp + upload a file

cd ~/swarm/scripts

./01-start-bee.sh                       # ~1–2 min to /readiness
eval "$(./02-buy-stamp.sh)"             # → BATCH=…
eval "$(./03-upload-file.sh)"           # → REF=… TESTFILE=…

Note the REF printed at the end — that's a 64-char hex hash you can retrieve via either local bee or zigbee.

A.5 Retrieve via zigbee

For a developer loop, point zigbee at your local bee instead of a public bootnode (zigbee will be connected to only bee, no other peers — useful for testing the SWAP path under controlled conditions):

./zig-out/bin/zigbee \
    --peer 127.0.0.1:1634 \
    --network-id 10 \
    --chequebook ~/.zigbee/chequebook.json \
    daemon --max-peers 1 --api-port 9090

# In another terminal:
curl -o /tmp/recovered.bin "http://127.0.0.1:9090/bytes/$REF"
cmp "$TESTFILE" /tmp/recovered.bin && echo "byte-identical ✅"

Or point zigbee at a public bootnode (production-shape) and let the network replicate your upload via bee's pushsync. Wait a minute or so after upload, then retrieve as in step 10 of the main guide.

A.6 Verification scripts

The scripts/ dir ships with three end-to-end verification scripts that exercise the same flow with assertions built in:

ScriptWhat it tests
./10-verify-zigbee-cache.sh "$REF" "$TESTFILE" Unencrypted retrieval via zigbee, plus the local LRU cache (cold/hot timing).
./11-verify-zigbee-encrypted-refs.sh "$REF128" "$TESTFILE" Encrypted-chunk references (128-character hex). Use ./04-upload-encrypted-file.sh to get a $REF128.
./12-verify-zigbee-swap.sh "$REF" "$TESTFILE" Full SWAP path — fires 25 retrievals, watches cheques fly, asserts bee accepted them. Requires the chequebook from step 7 of the main guide.

A.7 Stop everything

./99-stop-all.sh

SIGINT zigbee + bee, wait for clean exit, fall through to TERM/KILL on timeout.
